# FastAPI — Serving a Machine Learning Model

> Part of the `ml-from-scratch` project  
> Path: `05_production/fastapi/project.md`

---

## Project Goal

The goal of this project is to learn how to serve a machine learning model as a REST API
using FastAPI. Rather than running predictions inside a notebook, the model is exposed via
HTTP endpoints — meaning any client (a browser, a script, another application) can send
features and receive a prediction in return.

This is the first step toward production-ready ML: moving from a notebook experiment to a
deployed, callable service.

---

## Model Source

The model served by this API is the `LinearRegression` class built from scratch in:

```
02_classical_ml/01_linear_regression/
```

It was trained on the **sklearn diabetes dataset** (10 float features, continuous target)
and saved using joblib at:

```
02_classical_ml/01_linear_regression/models/linear_regression_scratch.pkl
```

Both the `.pkl` file and the `linear_regression.py` class definition were copied into
`05_production/fastapi/` so the API is self-contained and does not depend on files
outside its own directory.

## What Each File Does

### `main.py`

The core of the API. It creates the FastAPI application, loads the model once at startup,
and defines two endpoints:

- `GET /health` — returns `{"status": "ok"}` to confirm the API is running
- `POST /predict` — receives 10 features as JSON, runs the model, returns the prediction

### `schemas.py`

Defines the `PredictRequest` class using Pydantic. This class describes exactly what the
client must send: 10 float fields (`age`, `sex`, `bmi`, `bp`, `s1`–`s6`). FastAPI uses
this schema to automatically validate incoming requests — if a field is missing or the
wrong type, the API rejects the request with a clear error.

### `linear_regression.py`

The custom `LinearRegression` class. joblib needs this file to reconstruct the model
object when loading the `.pkl`. Without it, loading the model raises a
`ModuleNotFoundError`.

### `linear_regression_scratch.pkl`

The serialized trained model. Loaded once at startup using `joblib.load()` and reused
for every prediction request.

## Step-by-Step: How to Run the API

### 1. Activate your conda environment

```bash
conda activate your_env_name
```

Replace `your_env_name` with the name of the environment where FastAPI and uvicorn are
installed. You can check available environments with:

```bash
conda env list
```

### 2. Install dependencies (first time only)

```bash
pip install fastapi uvicorn joblib numpy
```

### 3. Navigate to the project folder

```bash
cd path/to/ml-from-scratch/05_production/fastapi
```

You must be in this folder when launching uvicorn — it looks for `main.py` in the
current directory.

### 4. Start the API

```bash
uvicorn main:app --reload
```

- `main` → the file `main.py`
- `app` → the FastAPI object defined inside it
- `--reload` → automatically restarts the server when you save changes (development only)

You should see:

```
INFO:     Uvicorn running on http://127.0.0.1:8000
INFO:     Application startup complete.
```

### 5. Test the `/health` endpoint

Open your browser and go to:

```
http://127.0.0.1:8000/health
```

Expected response:

```json
{ "status": "ok" }
```

### 6. Test the `/predict` endpoint via `/docs`

Open your browser and go to:

```
http://127.0.0.1:8000/docs
```

This is the automatic interactive documentation generated by FastAPI. Click on
`POST /predict`, then `Try it out`, paste the following JSON body, and click `Execute`:

```json
{
  "age": 0.038075906433423026,
  "sex": 0.05068011873981862,
  "bmi": 0.061696206518683294,
  "bp": 0.0218723855140367,
  "s1": -0.04422349842444599,
  "s2": -0.03482076283769895,
  "s3": -0.04340084565202491,
  "s4": -0.002592261998183278,
  "s5": 0.019907486170462722,
  "s6": -0.01764612515980379
}
```

Expected response:

```json
{ "prediction": 203.01935842396034 }
```

### 7. Stop the API

Press `CTRL+C` in the terminal.

## Key Concepts Learned

- A REST API exposes functionality over HTTP using standard verbs (GET, POST)
- FastAPI defines endpoints as Python functions decorated with `@app.get` or `@app.post`
- Pydantic schemas validate incoming data automatically before the function is called
- The model is loaded once at startup (not on every request) for performance
- uvicorn is the server that runs the FastAPI application on the network
- The API is self-contained — all files it needs live in its own directory

---

## How to Reuse This Pattern for Another Model

1. Train and save your model with joblib in your algorithm folder
2. Copy the `.pkl` and the class definition into `05_production/your_project/`
3. Update `schemas.py` to match your model's input features and types
4. Update `main.py` to load the new model and build the correct NumPy array
5. Run with `uvicorn main:app --reload` from the project folder

The structure stays the same — only the schema and the model change.
